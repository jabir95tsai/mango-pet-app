#### P2-2 `users/{uid}` update rule 無欄位白名單
- **問題**:`allow update: if request.auth.uid == uid;` 無 `hasOnly()` 約束,user 可往自己 doc 寫任意欄位/任意型別(垃圾欄位、超大 doc、偽 `isGuest`/`authProvider` 顯示值)。
- **證據**:`firestore.rules:47`。緩解:rules 的 `isGuest()` 是 token-based(`firestore.rules:24-27`),不受 doc 欄位影響;PII 已移出 → 影響限資料品質/顯示,非權限突破。
- **嚴重度**:P2。**交付**:Backend(rules 加欄位白名單 + 型別檢查;對齊 `pets`/`photoDownloadState` 已有的寫法)。**工作量**:S。**spec 重疊**:security-hardening 精神延伸,可掛同 spec 補一條 #5。

#### P2-3 Firestore 讀取全面 `doc.data() as X` 無 runtime 驗證(系統性)
